# 0.Inbox 缓存箱 (Inbox)

> 临时存储区，收集灵感和待处理信息

## 📍 定位

收件箱是整个知识管理系统的「入口缓冲区」，用于快速捕获任何想法、灵感、待办事项和临时信息。这是大脑「外接硬盘」，清空大脑缓存的地方。

**核心原则**：快速捕获，定期整理，绝不停留。

## 📂 存储内容类型

- **灵感闪现**：突然想到的项目点子、创意、思考
- **待办任务**：需要执行但尚未分类的任务
- **临时笔记**：不想立即归类的碎片信息
- **收集素材**：从外部阅读中摘录的内容
- **问题记录**：需要后续研究或回答的问题
- **会议速记**：临时记录的想法和要点

## 📏 命名规范

### 文件命名规则

```
YYYY-MM-DD-[简短描述].md
```

### 示例

```
2026-01-27-关于项目架构的想法.md
2026-01-26-待办事项清单.md
2026-01-25-阅读笔记碎片.md
```

### 命名要求

| 规则 | 说明 |
|------|------|
| 日期格式 | 使用 ISO 格式 `YYYY-MM-DD`，放在文件名前缀 |
| 分隔符 | 使用 `-` 连接各部分 |
| 描述长度 | 简短描述不超过 5 个词 |
| 字符限制 | 总文件名长度不超过 50 字符 |
| 禁止字符 | 不使用 `\/:*?"<>|` |

### TODO.md 特殊说明

- `TODO.md` 用于收集待办事项，每周整理
- 每条待办使用 `- [ ] ` 格式标记
- 定期将已确定的待办移动到相应项目或领域

## 🔄 整理周期

| 周期 | 动作 |
|------|------|
| 每日 | 查看收件箱，补充新内容 |
| 每周 | 整理本周内容，移动到对应文件夹 |
| 每月 | 清理过期内容，删除或归档 |

## ⚠️ 注意事项

1. **不要**在收件箱中长期存放内容（超过 2 周）
2. **不要**在收件箱中创建深层文件夹结构
3. **不要**追求完美，先捕获再优化
4. **保持**快速录入，不要中断思路
