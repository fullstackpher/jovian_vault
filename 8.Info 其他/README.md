# 8.Info 其他 (Miscellaneous)

> 不属于其他分类的杂项内容、实验性笔记、临时存放

## 📍 定位

其他文件夹是**兜底分类**，用于存放不属于标准分类体系的内容。这是一个「杂物间」，避免因为找不到合适的位置而随意放置文件。

**核心原则**：明确分类，定期整理，最终归位。

## 📂 存储内容类型

- **实验笔记**：尝试新方法、新工具的测试记录
- **草稿内容**：尚未定稿、正在构思的内容
- **杂项**：确实无法归类但可能有价值的内容
- **临时存放**：不确定归属，先放这里的文件
- **个人内容**：不便归入其他分类的个人笔记
- **未整理**：需要后续整理但目前无暇顾及的内容

## 📏 命名规范

### 命名规则

```
[类型]-[简短描述].md
```

### 示例

```
# 实验性质
实验-Obsidian新插件测试.md
实验-Mermaid新语法尝试.md

# 草稿内容
草稿-关于AI学习的思考.md
草稿-项目架构设计-V1.md

# 杂项
杂项-常用链接汇总.md
杂项-账号密码记录.md
杂项-订阅服务清单.md

# 临时存放
临时-待整理的笔记.md
临时-需要分类的文章.md
```

### 命名要求

| 规则 | 说明 |
|------|------|
| 类型前缀 | 使用 `实验-`、`草稿-`、`杂项-`、`临时-` 等前缀 |
| 描述简洁 | 简短描述不超过 10 个词 |
| 避免字符 | 不使用 `\/:*?"<>|` |
| 加锁标记 | 敏感信息文件可加密处理 |

### 类型前缀对照表

| 前缀 | 类型 | 说明 |
|------|------|------|
| `实验-` | 实验笔记 | 新工具、新方法的测试记录 |
| `草稿-` | 草稿内容 | 尚未完成的构思 |
| `杂项-` | 杂项内容 | 无法归类的零散内容 |
| `临时-` | 临时存放 | 等待后续整理的内容 |
| `私密-` | 私人内容 | 不便公开的个人信息 |
| `WIP-` | 进行中 | 正在进行的工作 |

## 🔄 整理周期

| 周期 | 动作 |
|------|------|
| 每日 | 新内容及时添加类型前缀 |
| 每周 | 检查「临时」文件夹，尝试归类 |
| 每月 | 审视「实验」和「草稿」内容，整理或删除 |
| 每季 | 大整理，清理过期内容 |

## ⚠️ 注意事项

1. **不要**将「其他」作为长期存放地
2. **不要**随意丢弃可能有价值的内容
3. **保持**类型前缀清晰，便于识别和处理
4. **定期**审视和整理，避免变成垃圾场
5. **敏感信息**使用 Obsidian 加密或密码保护

## 💡 整理策略

### 进出平衡原则

- **进**：只有暂时无法归类的内容才能进入
- **出**：定期整理，将内容移至合适的分类
- **目标**：保持「其他」文件夹的内容数量在可控范围

### 内容处理流程

```
新内容 → 评估能否归类
  │
  ├─ 能归类 → 移到对应文件夹
  │
  └─ 不能归类 → 放入「其他」，添加类型前缀
                │
                ↓
            定期整理
                │
                ├─ 可归类 → 移到对应文件夹
                ├─ 有价值 → 完善后归类
                └─ 无价值 → 删除
```

### 常见处理方式

| 原内容 | 最终归属 |
|--------|----------|
| 新工具测试记录 | 资源-工具教程 |
| 新方法实验结果 | 领域-最佳实践 |
| 完成的草稿文章 | 可以独立发布或移到对应领域 |
| 临时链接收集 | 资源-链接收藏 |
| 过时实验记录 | 删除或归档 |

## 🔗 与其他目录的连接

- **收件箱**：「其他」内容的来源之一
- **领域**：实验结果可能发展为领域知识
- **资源**：工具测试可能发展为资源文档
- **项目**：草稿可能发展为项目内容

## 💡 使用建议

### 1. 优先使用标准分类

在放入「其他」之前，先思考：
- 这个内容属于哪个领域？
- 这个内容属于哪个项目？
- 这个内容是资源还是工具？

### 2. 使用标签辅助管理

```markdown
---
tags: [ #实验, #草稿, #待整理 ]
---

# 内容标题
```

### 3. 定期回顾机制

建议在每月复盘时顺便检查「其他」文件夹：
- 这个月放进来多少内容？
- 有多少已经移出或删除？
- 有没有应该归类但还没处理的内容？

## 📊 理想状态

长期目标是使「其他」文件夹：
- ✅ 内容数量保持在较低水平（< 20个文件）
- ✅ 所有内容都有清晰的类型前缀
- ✅ 没有存放超过 3 个月的临时内容
- ✅ 定期有人整理和清理
