> [!NOTE] CSS相对单位
> CSS中的相对单位是相对于其他值来定义的，这使得布局更加灵活且能够适应不同大小的屏幕。以下是一些常用的相对单位：
> 
> 1. **em** - 相对于元素的字体大小（font-size）。如果一个元素没有设置字体大小，则会参考其父元素的字体大小。例如，如果一个元素的字体大小为16px，那么1em就等于16px。
>    - 用法示例：`font-size: 2em;` 表示字体大小为当前元素或其最近设置了字体大小的祖先元素字体大小的两倍。
> 
> 2. **rem $root em$** - 类似于em，但总是相对于根元素（即HTML标签）的字体大小。这样可以确保在整个文档中保持一致的比例关系。
>    - 用法示例：假设`<html>`标签的字体大小设为了16px，那么无论在哪一层级使用`font-size: 1rem;`都会等于16px。
> 
> 3. **% $百分比$** - 可以应用于多种属性如宽度、高度等，并且总是相对于包含块的尺寸。对于字体大小而言，它基于父元素的字体大小。
>    - 用法示例：`width: 50%;` 将使元素占据其直接父元素宽度的一半。
> 
> 4. **vw $viewport width$** 和 **vh $viewport height$** - 分别代表视口宽度和高度的百分之一。这些单位非常适合创建响应式设计。
>    - 用法示例：`width: 100vw;` 会让元素宽度等于整个浏览器窗口的宽度；而`height: 50vh;` 则设置元素高度为视口高度的一半。
> 
> 5. **vmin** 和 **vmax** - vmin选取视口宽度和高度中较小的那个值作为基准，然后取其百分之一；vmax则相反，它选取较大者。
>    - 用法示例：`font-size: 2vmin;` 字体大小将根据视口中较短边长的2%来计算。
> 
> 使用相对单位有助于创建更灵活的设计方案，尤其是在需要适应不同设备和屏幕尺寸的情况下。通过合理地结合使用这些单位，你可以构建出既美观又实用的网页布局。

> [!NOTE] 视口单位应用场景
> 视口单位（Viewport units）是CSS中的一种相对长度单位，包括vw（viewport width）、vh（viewport height）、vmin（viewport minimum）和vmax（viewport maximum）。这些单位根据浏览器窗口的尺寸来定义元素的大小，使得页面布局更加灵活、响应式。以下是视口单位的一些常见应用场景：
> 
> 1. **全屏背景图片或颜色**：
>    - 使用`100vw` 和 `100vh` 可以让背景覆盖整个视窗区域。
>    - 例如：设置一个全屏背景图片可以使用如下CSS代码：
>      ```css
>      .fullscreen-bg {
>        width: 100vw;
>        height: 100vh;
>        background-image: url$'background.jpg'$;
>        background-size: cover;
>      }
>      ```
> 
> 2. **响应式字体大小**：
>    - 视口单位可以用来创建随视窗大小变化而调整的字体大小。
>    - 例如：标题字体大小可以根据屏幕宽度进行调整：
>      ```css
>      h1 {
>        font-size: 5vw; /* 根据视窗宽度的5%来设置字体大小 */
>      }
>      ```
> 
> 3. **响应式布局**：
>    - 在构建响应式网页时，视口单位可以帮助实现更流畅的布局调整。
>    - 例如：创建一个响应式的导航栏，使其在不同设备上都能良好显示：
>      ```css
>      .navbar {
>        width: 100%;
>        height: 10vh; /* 导航栏高度为视窗高度的10% */
>        position: fixed;
>        top: 0;
>        left: 0;
>        background-color: #333;
>      }
>      ```
> 
> 4. **自适应间距**：
>    - 使用视口单位可以创建随视窗大小变化的间距，从而改善布局的视觉效果。
>    - 例如：设置段落之间的间距：
>      ```css
>      p {
>        margin-bottom: 2vh; /* 段落底部间距为视窗高度的2% */
>      }
>      ```
> 
> 5. **复杂的网格系统**：
>    - 结合CSS Grid或Flexbox，视口单位可以用于创建复杂的、自适应的网格布局。
>    - 例如：创建一个三列布局，每列宽度随视窗变化：
>      ```css
>      .grid-container {
>        display: grid;
>        grid-template-columns: 1fr 1fr 1fr;
>        gap: 2vw; /* 列间距为视窗宽度的2% */
>      }
>      ```
> 
> 6. **响应式按钮和表单控件**：
>    - 视口单位可以用于创建响应式的按钮和表单控件，使其在不同设备上都能保持良好的可用性。
>    - 例如：创建一个响应式的按钮：
>      ```css
>      .responsive-button {
>        padding: 1vh 2vw; /* 上下内边距为视窗高度的1%，左右内边距为视窗宽度的2% */
>        font-size: 2vw; /* 字体大小为视窗宽度的2% */
>      }
>      ```
> 
> 通过这些应用场景，视口单位可以帮助开发者创建更加灵活、适应性强的网页布局，提升用户体验。
