### 📦 第一步：搭建PARA基础文件夹结构
先在Obsidian的根目录下创建4个核心文件夹，这是整个系统的骨架：

- **Projects（项目）**：存放有明确目标和截止日期的短期任务集合，比如「开发Flutter个人APP」「整理Java后端笔记」，每个项目单独建一个子文件夹，里面放项目相关的所有笔记、任务清单、参考资料
- **Areas（领域）**：存放需要长期维护的领域知识，比如「前端技术」「后端技术」「投资理财」，按你关注的领域分类，每个领域下再细分具体主题
- **Resources（资源）**：存放碎片化的参考资料，比如从网页复制的技术文档、收藏的行业报告、UP主分享的Obsidian技巧，按主题打标签，方便后续检索复用
- **Archives（归档）**：存放已完成的项目、不再关注的领域内容，比如「2025年二手苹果产品调研」，定期把Projects里完成的项目移到这里，保持系统清爽

### 🎯 第二步：融入GTD的行动管理流程

在PARA的框架上叠加GTD的5个核心步骤，让知识和行动关联起来：

- **收集**：在根目录创建一个`Inbox`（收件箱）文件夹，作为临时收集箱，所有想法、待办、灵感都先丢这里，比如突发的APP功能idea、待整理的技术笔记、投资理财的新观点，避免信息遗漏
- **整理**：每天固定时间（比如晚上10点）处理Inbox里的内容：
    - 如果是**行动项**：比如「学习Flutter状态管理」，放到对应的Projects或Areas文件夹，并用`- [ ]`标记为任务，还可以用`#任务/待办`标签分类
    - 如果是**知识内容**：比如「Java集合框架总结」，归类到Areas下的「后端技术」文件夹
    - 如果是**参考资料**：比如「Obsidian插件推荐清单」，放到Resources文件夹，打上`#工具/Obsidian`标签
- **组织**：用Obsidian的标签和链接功能串联内容：
    - 给任务笔记添加`#项目/FlutterAPP` `#领域/前端`这样的标签，方便跨文件夹检索
    - 在项目笔记里用`[[前端技术/Flutter状态管理]]`链接到对应的领域知识笔记，实现行动和知识的联动
- **执行**：用**Tasks插件**创建任务看板，在根目录建一个`任务看板.md`，输入查询语句：

```markdown
	```tasks 
	不包含 #归档 排序方式为优先级，日期
	
	这样就能看到所有待办任务，按优先级和截止日期排序，每天打开这个笔记就能直接开始行动
	```
```

- **回顾**：每周日做一次周回顾，打开`任务看板.md`检查任务完成情况，把已完成的任务标记为`- [x]`，并移到Archives文件夹；同时更新Projects里的项目进度，调整Areas里的领域知识内容

### 🧩 第三步：用插件增强系统效率

搭配几个Obsidian插件，让GTD+PARA系统更丝滑：

- **Templater**：创建模板笔记，比如「项目模板.md」「领域笔记模板.md」，里面预设好标题、任务清单、参考链接的格式，新建笔记时直接套用，节省整理时间
- **Dataview**：制作知识仪表盘，在根目录建一个`知识仪表盘.md`，用Dataview查询不同文件夹的内容：
```markdown
	```dataview TABLE file.mtime AS "最后修改时间" FROM "Projects" SORT file.mtime DESC 
	
	
	这样就能直观看到所有项目的更新情况，不用一个个文件夹打开查看
	```
```

- **Calendar**：在侧边栏添加日历，点击日期直接创建当天的任务笔记，比如「2026-02-01 任务清单.md」，把当天要做的事都记录在这里，和GTD的执行环节完美契合

### 📝 第四步：日常使用流程示例

给你举个具体的使用场景，帮你快速上手：

1. 看到一篇关于Flutter性能优化的文章，先复制到Inbox文件夹，命名为「Flutter性能优化技巧.md」
2. 晚上整理Inbox时，判断这是前端领域的知识内容，移到Areas下的「前端技术」文件夹，打上`#领域/前端` `#技术/Flutter`标签
3. 同时想到APP开发需要用到这个技巧，在「开发Flutter个人APP」项目笔记里添加任务`- [ ] 学习Flutter性能优化技巧`，并链接到这篇笔记：`[[前端技术/Flutter性能优化技巧]]`
4. 第二天打开「任务看板.md」，看到这个任务，点击链接直接学习，学完后把任务标记为完成，在笔记里补充自己的理解，让知识真正沉淀下来
