# 5.Zattchments 附件 (Attachments)

> 项目相关的非文本文件、媒体资源、参考资料

## 📍 定位

附件文件夹用于存放笔记中引用到的**非Markdown文件**，包括图片、PDF、音频、视频等媒体资源。这是整个知识库的「资源仓库」，保持笔记目录的清洁。

**核心原则**：集中管理，链接引用，定期清理。

## 📂 存储内容类型

- **截图**：日常工作中截取的图片
- **PDF文档**：电子书、论文、技术文档等
- **设计稿**：UI设计图、流程图、架构图
- **幻灯片**：演示文稿、课程PPT
- **代码文件**：代码片段、配置文件（不适合放笔记的内容）
- **媒体文件**：音频、视频等（如果需要引用）
- **项目资源**：项目专属的静态资源

## 📏 命名规范

### 文件命名规则

```
[YYYY-MM-DD]-[简短描述].[扩展名]
```

### 示例

```
2026-01-30-系统架构图.png
2026-01-29-会议记录.pdf
2026-01-28-设计稿-v1.psd
2026-01-27-代码片段.sql
```

### 命名要求

| 规则 | 说明 |
|------|------|
| 日期前缀 | 使用 ISO 格式 `YYYY-MM-DD` 放在文件名前缀 |
| 分隔符 | 使用 `-` 连接各部分 |
| 描述简洁 | 简短描述不超过 5 个词 |
| 扩展名 | 保留原始文件扩展名 |
| 禁止字符 | 不使用 `\/:*?"<>|` |

## 🔗 在笔记中引用

### 图片引用

```markdown
![描述](图片/截图/2026-01-30-截图.png)
```

或使用 Obsidian 嵌入语法：

```markdown
![描述](5.Zattchments 附件/图片/截图/2026-01-30-截图.png)
```

### PDF 引用

```markdown
[文档标题](文档/书籍/书名.pdf)
```

### 推荐使用嵌入

对于图片，推荐使用 Obsidian 的嵌入语法：

```markdown
![[图片/截图/2026-01-30-截图.png]]
```

这样可以直接在笔记中预览图片。

## 🔄 整理周期

| 周期 | 动作 |
|------|------|
| 每日 | 截图随手保存，命名规范化 |
| 每周 | 检查临时文件，清理无用资源 |
| 每月 | 审视附件结构，归档旧资源 |

## ⚠️ 注意事项

1. **不要**在附件文件夹中创建笔记文件（.md）
2. **不要**直接粘贴大文件到附件目录，使用引用方式
3. **保持**文件命名规范，便于搜索和整理
4. **定期**清理不再使用的附件，保持仓库轻量
5. **注意**大文件对仓库同步的影响

## 💡 与资源的区别

| 附件 | 资源 |
|------|------|
| 非文本文件 | 可编辑的文档/笔记 |
| 图片、PDF、音视频 | 工具教程、代码片段 |
| 被笔记引用 | 独立可用的知识 |
| 二进制格式 | 纯文本格式 |
