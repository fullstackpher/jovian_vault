### 认识JavaScript变量
- JS的变量是存储数据的容器，它存放的数据都是可变的

### 变量的定义、规范

> [!info] 在JS中命名变量，包含两部分：

- 变量的声明：在JS中声明变量使用`var`关键字，ES6中使用let、const方式
- 变量的赋值：使用 = 给变量进行赋值

这个过程也可以分开操作：

```js
var name;
name = 'jovian';
```

同时声明多个变量：

```js
var name, age, sex;
name = "Alice";
age = 25;
sex = "Female"
console.log(name, age, sex);
```

>[!danger] 变量的命名规范
>	变量的命名规则：必须遵守

- 首字母必须是一个字母、下划线（ _ ）或一个美元符号（ $ ）
- 其他字符可以是字母、下划线、美元符或数字
- 不能使用**关键字**或**保留字**命名
	- [关键字/保留字](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Lexical_grammar)
- 变量**严格区分大小写**
- 多个单词使用**驼峰标识**
- 变量应该做到**见名知意**

>[!danger] 变量的使用注意

- 注意一：如果一个变量未声明（declaration）就直接使用，那么会报错
	- **Uncaught ReferenceError: message is not defined**
	- `name`是全局属性，直接输出name可能为空
- 注意二：如果一个变量有声明，但没有赋值，那么默认就是undefined
- 注意三：如果没有使用var声明变量，但不推荐这种定义方式（事实上会被加到window对象上）

### 数据类型和typeof

>[!info] JavaScript中的值都具有特定的类型的
-

- 我们将值赋给一个变量，这个变量就具备了特定的类型
- 一个变量前一刻是个字符串，下一刻就变成了一个数字
- 允许这种操作的编程语言，例如JavaScript，被称为“动态类型”的编程语言

>[!tip] JS中有8种基本的数据类型

- Number
- String
- Boolean
- Undefined
- Null
- Object
- BigInt
- Symbol

>[!info] typeof操作符

- 因为ECMAScript的类型系统是松散的，所以需要一种手段来确定任意变量的数据类型
	- typeof操作费就是为此而生的
- 对一个值使用typeof操作符会返回如下字符串之一：
	- "undefined"表示值未定义
	- "boolean"表示值为布尔值
	- "string"表示值为字符串
	- "number"表示值为数值
	- "object"表示值为对象（而不是函数）或null
	- "function"表示值为函数
	- "symbol"表示值为符号
- typeof()的用法：
	- `typeof(x)`与`typeof x`是一样的
	- typeof是一个**操作符**，**并非是一个函数，()只是将后续的内容当做一个整体而已**

### 常见的数据类型
#### Number
- number类型代表整数和浮点数
- 数字number可以有很多操作，比如乘法，除法，加法，减法，等等
- 除了常规的数字，还包括所谓的“特殊数值”也属于Number类型
	- **Infinity**：代表数学概念中的无穷大，也可以表示-Infinity
		- 比如1/0得到的就是无穷大
	- **NaN**：NaN代表一个计算错误，它是一个错误的操作所得到的结果
		- 比如字符串和一个数字相乘
- 数字类型关于进制的表示方法
	- **十进制**（常用）
	- 十六进制
	- 二进制
	- 八进制
- 数字表示的范围
	- 最小正数值：Number.MIN_VALUE，这个值是：5e-324，小于这个数会被转化为0
	- 最大正数值：Number.MAX_VALUE，这个值是：1.7976931348623157e+308
- isNaN
	- 用于判断是否为一个数字，**不是数字返回true，是数字返回false**
#### String
- 在开发中，经常会有一些文本要表示，这个时候就会使用字符串String
	- 比如人的姓名：Jovian，地址：深圳市。简介：认真是一种可怕的力量
- JavaScript中的字符串必须被括在引号内，有三种包含字符串的方式
	- 双引号`“Hello”`
	- 单引号`’Hello‘`
	- 反引号 
	```js
	var info = `hello`;
	```
