# 03-资源 (Resources)

> 工具教程、参考文献、最佳实践库

## 📍 定位

资源文件夹是**可复用的知识资产库**，存放各种工具的使用指南、参考资料、代码片段和最佳实践。与领域知识不同，资源更偏向于「工具属性」——即查即用，快速上手。

**核心原则**：即查即用，分类清晰，持续更新。

## 📂 存储内容类型

- **工具教程**：开发工具、效率工具的使用指南
- **快捷速查**：命令速查、API 速查、配置速查
- **代码片段**：常用代码模板、可复用函数
- **配置参考**：.gitignore、IDE 配置、Dockerfile 等
- **最佳实践**：编程规范、安全实践、性能优化
- **图书笔记**：读书笔记、课程笔记
- **链接收藏**：优质文章、教程、网站的摘要和链接

## 📏 命名规范

### 命名规则

```

```

### 命名要求

| 规则 | 说明 |
|------|------|
| 文件名称 | 使用中文或英文，保持简洁直观 |
| 避免冗余 | 不使用「使用指南」「教程」等冗余词 |
| 保持一致 | 同类资源使用统一的命名风格 |
| 禁止字符 | 不使用 `\/:*?"<>|` |

### 类型前缀对照表（可选）

| 前缀 | 类型 | 示例 |
|------|------|------|
| `工具-` | 开发工具 | 工具-Visual Studio Code.md |
| `速查-` | 速查手册 | 速查-Git命令.md |
| `规范-` | 规范文档 | 规范-代码规范.md |

## 🔍 使用方式

1. **遇到问题** → 在资源库中搜索解决方案
2. **学习新工具** → 先阅读对应的工具教程
3. **写代码时** → 参考代码片段和最佳实践
4. **发现好资源** → 及时整理添加到资源库

## ⚠️ 注意事项

1. **不要**将资源与领域知识混淆（工具教程放这里，领域知识放领域）
2. **不要**直接复制大段内容，提取关键点
3. **保持**内容简洁，突出实用性
4. **定期**清理过时内容，保持资源库质量

## 💡 与领域的区别
