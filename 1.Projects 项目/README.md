# 01-项目 (Projects)

> 正在执行的具体项目，有明确目标和截止日期

## 📍 定位

项目文件夹用于存放**正在进行中**的独立工作单元。每个项目应有明确的起止时间、具体目标和可交付成果。这是执行层面的核心区域。

## 📂 存储内容类型

- **项目主文件**：项目规划、进度追踪、决策记录
- **需求文档**：功能需求、用户故事、规格说明
- **开发记录**：代码实现、技术选型、问题解决
- **会议纪要**：项目相关会议讨论要点
- **测试文档**：测试用例、测试结果、Bug 记录
- **交付物**：项目产出的文档、代码、设计稿

## 📏 命名规范

### 项目文件夹命名

```
NN-[项目名称]/
```

### 项目文件命名

```
NN-[项目名称]/[类型]-[描述].md
```

### 示例

```
01-学习Java/
├── 00-项目规划.md
├── 01-学习路线图.md
├── 02-笔记索引.md
└── 进度追踪.md

02-知识管理/
├── 需求分析.md
├── 系统设计.md
└── 开发日志.md
```

### 命名要求

| 规则 | 说明 |
|------|------|
| 前缀编号 | 使用两位数字 `01-`, `02-` 等排序 |
| 项目名称 | 使用中文描述，不超过 10 个字 |
| 文件类型前缀 | `00-` 核心文件，`01-` 规划，`02-` 笔记等 |
| 禁止字符 | 不使用 `\/:*?"<>|` 和空格 |

### 类型前缀对照表

| 前缀 | 类型 | 说明 |
|------|------|------|
| `00-` | 项目核心 | 项目规划、README 等入口文件 |
| `01-` | 规划类 | 学习路线、需求分析、设计文档 |
| `02-` | 笔记类 | 学习笔记、技术记录 |
| `03-` | 追踪类 | 进度追踪、Bug 记录 |
| `04-` | 交付物 | 完成的文档、代码片段 |
| `XX-` | 临时类 | 会议纪要、临时讨论 |

## 🏗️ 项目结构模板

```
[NN-项目名称]/
├── 00-项目规划.md          # 项目目标、范围、时间线
├── 01-需求文档.md          # 详细需求说明
├── 02-技术方案.md          # 技术选型、架构设计
├── 03-开发日志.md          # 每日/周开发进展
├── 04-测试记录.md          # 测试用例和结果
└── 附件/                   # 图片、代码等资源
```

## 🔄 项目生命周期

| 阶段 | 操作 |
|------|------|
| 启动 | 在收件箱中提取，创建项目文件夹 |
| 执行 | 按命名规范创建子文件，记录进展 |
| 完成 | 整理文档，移动到 `04-归档` |
| 暂停 | 记录暂停原因，移动到 `04-归档` |

## ⚠️ 注意事项

1. **每个项目**必须有一个 `00-项目规划.md` 作为入口
2. **保持**项目文件夹扁平，避免超过 2 层嵌套
3. **定期**更新进度追踪，至少每周一次
4. **完成后**必须整理并归档，不留在项目文件夹
