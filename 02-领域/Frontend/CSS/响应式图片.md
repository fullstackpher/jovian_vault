> [!NOTE] 响应式图片
> 响应式图片是指能够根据用户设备的屏幕尺寸、分辨率和方向自动调整大小和质量的图片。这种技术可以确保网页在不同设备上都能提供良好的用户体验，同时还能节省带宽，提高页面加载速度。下面是一些实现响应式图片的方法和技术：
> 
> ### 1. 使用HTML5的`<picture>`元素
> `<picture>`标签允许开发者为不同的屏幕尺寸指定不同的图片源。这比简单地使用CSS媒体查询更加灵活，因为它还考虑了图片格式的支持情况。
> 
> ```html
> <picture>
>   <source media="$min-width: 1200px$" srcset="large.jpg">
>   <source media="$min-width: 768px$" srcset="medium.jpg">
>   <img src="small.jpg" alt="Responsive Image">
> </picture>
> ```
> 
> ### 2. 利用`srcset`属性
> `srcset`属性允许你列出多个图像文件及其对应的视口宽度或像素密度。浏览器会根据当前屏幕的实际条件选择最合适的版本进行显示。
> 
> ```html
> <img src="default.jpg"
>      srcset="small.jpg 300w,
>              medium.jpg 600w,
>              large.jpg 900w"
>      sizes="$max-width: 300px$ 100vw,
>             $max-width: 600px$ 50vw,
>             33vw"
>      alt="Responsive Image">
> ```
> 
> - `300w`, `600w`, `900w`：表示每个图片资源的固有宽度。
> - `sizes`属性定义了不同屏幕尺寸下的图片显示宽度。
> 
> ### 3. CSS背景图片
> 对于作为背景使用的图片，可以通过CSS媒体查询来控制不同断点下的图片路径。
> 
> ```css
> .background {
>   background-image: url$'default.jpg'$;
> }
> 
> @media $min-width: 600px$ {
>   .background {
>     background-image: url$'medium.jpg'$;
>   }
> }
> 
> @media $min-width: 1200px$ {
>   .background {
>     background-image: url$'large.jpg'$;
>   }
> }
> ```
> 
> ### 4. 图片懒加载
> 为了进一步优化性能，特别是对于长滚动页面，可以采用懒加载技术——只有当图片即将进入可视区域时才开始加载。
> 
> ```html
> <img data-src="image.jpg" class="lazyload" alt="Lazy Loaded Image">
> 
> <script>
>   document.addEventListener$"DOMContentLoaded", function$$ {

> ```
